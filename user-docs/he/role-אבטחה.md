# מדריך לאבטחה

מדריך זה מיועד ל**ראש אבטחה** ולצוות הביטחון. הוא מכסה את מודול **אבטחה** —
**יומן אירועי ביטחון**.

לפני שממשיכים — קראו את [01-התחלה](01-התחלה.md) ואת
[02-תפקידים-והרשאות](02-תפקידים-והרשאות.md).

---

## מה המודול עושה

מודול **אבטחה** (`/dashboard/security`) הוא **יומן אירועי ביטחון**: תיעוד,
מעקב וטיפול באירועים.

## רשימת האירועים

- **«+ אירוע חדש»** (לפי הרשאת ניהול) פותח טופס: כותרת (חובה), תיאור, מועד
  התרחשות, בניין, מיקום חופשי, קטגוריה, וחומרה (ברירת מחדל: בינונית).
- **סרגל סיכום**: מונים לכל סטטוס (פתוח / בחקירה / טופל / סגור), והדגשת «פעילים»
  ו«קריטיים».
- **מסננים**: סטטוס, חומרה, קטגוריה. **ייצוא CSV**.
- שורות האירוע מציגות כותרת, קטגוריה, מיקום, תאריך, ותגיות חומרה/סטטוס. אירוע
  קריטי/גבוה מודגש. לחיצה עוברת לפרטים.

## פרטי אירוע

- כרטיס מידע: חומרה/סטטוס/קטגוריה, מיקום, מועד, מועד טיפול, אחראי, תיאור, ופתרון.
- **פעולות** (לפי הרשאת ניהול):
  - **שינוי סטטוס** — רק מעברים מותרים (פתוח ← בחקירה/סגור; בחקירה ← טופל/סגור;
    טופל ← סגור/בחקירה; «סגור» סופי). מעבר ל«טופל» שולח גם את טקסט הפתרון.
  - **קביעת חומרה** — נמוכה / בינונית / גבוהה / קריטית.
  - **שיוך** — «שייך אליי» / «בטל שיוך».
  - **פתרון** — שדה טקסט + שמירה.

## טיפים

- השתמשו בחומרה כדי לתעדף — «קריטי» בולט בסרגל הסיכום.
- חיפוש מהיר: `Ctrl/Cmd + K`.
