# מכון חמש — ארכיטקטורת המערכת (גרסה 2)

שפות: **רוסית (ראשית)** + אנגלית + עברית.
מנהל על: **הרב אברהם** — רואה ושולט בהכל.

> המסמך הזה הוא ה"ליבה". כל מה שקיים במערכת הנוכחית מעבר לזה = בונוס להשוואה.

---

## 1. היררכיה ניהולית

ראה תרשים: `machon_chamesh_hierarchy.mermaid`

```
המנהל — הרב אברהם
├── אחראי לימודי קודש → אדמיניסטרטור/מזכירות → מורים → תלמידות
├── מנהלת התיכון (= בית ספר) → סגנית → מזכירות → מורים → תלמידות
├── אחראית קולג׳ (מקצוע) → סגנית → מזכירות → מורים → תלמידות
├── רקטור (אוניברסיטה) → דיקן → מזכירות → מרצים → תלמידות
├── מנהל הטורו → סגן → מזכירות → מורים → תלמידות
└── מנהלת אמונה → [תוכנית עתידית — תשתית בלבד]
```

---

## 2. אנטומיה של "מערכת" (התבנית שחוזרת בכל ענף)

כל ענף הוא **מערכת עצמאית** שמנהלת את עצמה. המבנה הפנימי הגנרי:

| תפקיד | מה עושה |
|---|---|
| ראש המערכת | אחראי/מנהל/מנהלת/רקטור — הובלה |
| סגן/ית | משנה, איפה שקיים |
| **מזכירות** | המנוע התפעולי: פותחת **כיתות**, קובעת **רמה**, משבצת **שיעורים** (שיבוצים), משייכת **תלמידות** |
| מורים / מרצים | משובצים לשיעור + כיתה. **מסמנים נוכחות**. גישה מצומצמת — רואים רק לאן להגיע |
| תלמידות | צרכניות הקצה — יומן, היסטוריה, דשבורד |

*אוניברסיטה = המערכת העמוקה ביותר (רקטור → דיקן → מזכירות → מרצים).*

## 3. מסלולים לימודיים

```
קטנות (מגיעות לכיתה י׳–י״א)
├── בית ספר  — שנתיים → בגרות → פתוח לאוניברסיטה   → מנוהל ע"י: מערכת התיכון
└── קולג׳    — 4 שנים → ידע מקצועי (לא תואר)        → מנוהל ע"י: מערכת הקולג׳

גדולות (אחרי בגרות)
├── מכללה     — 3 שנים                              → מנוהל ע"י: מערכת הקולג׳ (אותה מערכת)
└── אוניברסיטה                                       → מנוהל ע"י: מערכת האוניברסיטה
    └── טורו  — תואר רוסי (+ אמריקאי במקביל)         → מנוהל ע"י: מערכת הטורו (ענף נפרד)

לימודי בוקר — קודש
└── כל הבנות, בכל מסלול. מחולק לקטנות/גדולות + רמות  → מנוהל ע"י: מערכת הקודש
```

**נקודת מפתח:** כל תלמידה שייכת ל־**שתי** מערכות בו-זמנית —
המערכת הלימודית שלה (בית ספר / קולג׳ / אוניברסיטה / טורו) **וגם** מערכת הקודש של הבוקר.
שני לוחות הזמנים מתמזגים ליומן אחד.

## 4. מודל תפקידים והרשאות

- **הרשאות לפי תפקיד ודרגה.** לא לפי אדם — לפי תפקיד.
- מגדירים **תפקיד** → משבצים אליו איש צוות שנרשם ב"רישום איש צוות".
- איש צוות אחד יכול להחזיק **כמה תפקידים** → המנהל מסדר את השיבוץ.
- אפשר לשבץ ישר בעת הרישום, או להשאיר ולשבץ אחר כך.
- **האצלה:** המנהל יכול לתת לעובד לא רק הרשאות, אלא גם את היכולת **לאשר הרשאות לכפוף שלו**.

---

## 5. מודל התלמידה

- **כניסה:** נפתחת רק אחרי מעבר **ועדת קבלה** ורישום כתלמידה. (מנגנון סיסמה — לשלב הבא.)
- **יומן** בסגנון Google Calendar עם התראות על משימות.
- **תצוגת צבעים ביומן הגדול:** כל יום מקבל צבע לפי הנוכחות (ראה סעיף 6). משבצת שעה = משבצת שיעור.
- **כניסה ליום ספציפי:** פירוט לכל שיעור — נוכחות/איחור/היעדרות, שם המורה, ותוכן השיעור.
- **דשבורד:** מספר שיעורים מתחילת השנה, אחוז נוכחות, ממוצע ציונים.
- **פגישות:** מורה קובע פגישה עם תלמידה אחרי השיעור → מופיע ביומן שלה → ניתן לסמן "בוצע".
- **חברות (שאיפה / עתידי):** לראות מי החברות שלה — **בלי** מידע עליהן.

## 6. חוקי נוכחות

| צבע | משמעות |
|---|---|
| 🟢 ירוק | נוכח |
| 🟠 כתום | איחור |
| 🔴 אדום | חיסור |

**כלל:** איחור = חצי חיסור. **שני איחורים = חיסור מלא.**
זהו הבסיס לחישוב אחוזי הנוכחות בדשבורד.

**זרימת הנתונים:** המזכירות משבצת שיעור (מורה + כיתה + זמן) → המורה מגיע ומסמן נוכחות → הסימון מזין את היסטוריית התלמידה, את צבעי היומן, ואת אחוזי הדשבורד.

## 7. ישויות עיקריות (טיוטה — לשלב הבא, לצורך השוואה למסד הנתונים)

- **מערכת** (Unit) — קודש / תיכון / קולג׳ / אוניברסיטה / טורו / אמונה
- **תפקיד** (Role) → הרשאות
- **איש צוות** (Staff) → מחזיק תפקיד אחד או יותר
- **כיתה** (Class) — שייכת למערכת, בעלת רמה
- **רמה** (Level)
- **תלמידה** (Student) — משויכת למערכת לימודית + למערכת קודש
- **שיעור** (Lesson) — מורה + כיתה + זמן + תוכן
- **נוכחות** (Attendance) — נוכח / איחור / חיסור
- **פגישה** (Meeting) — מורה ↔ תלמידה, ניתנת לסימון "בוצע"
