3. **staging + Playwright E2E:** פרויקט Supabase נפרד עם עותק של סכימת
   הפרודקשן, ותרחישי דפדפן ראשונים (התחברות, אימות) שהוכחו מולו.

כך נראה קובץ ה-CI — פשוט וקריא:


### ג. טעימה מהקוד — איך נראית העבודה מבפנים

דוגמה אמיתית מהיום, מלב מודול המעונות: הלוגיקה של תפוסת חדרים נכתבה
כפונקציות "טהורות" — בלי שום גישה למסד — ולכן קל לבדוק אותה אוטומטית:

```ts
// lib/dormitory/occupancy.ts — חישוב תפוסת חדר, לוגיקה טהורה

/** האם שיבוץ פעיל בתאריך נתון (טווח פתוח = ללא תאריך סיום). */
export function isActiveOn(a: Assignment, dateISO: string): boolean {
  return (
    a.status === 'active' &&
    a.assigned_from <= dateISO &&
    (a.assigned_to === null || a.assigned_to >= dateISO)
  )
}

/** תפוסת חדר בתאריך נתון, לפי רשימת השיבוצים שלו. */
export function occupancy(assignments: Assignment[], capacity: number, dateISO: string) {
  const occupied = assignments.filter(a => isActiveOn(a, dateISO)).length
  const free = Math.max(0, capacity - occupied)
  return { capacity, occupied, free, isFull: occupied >= capacity }
}
```

וכך נראות הבדיקות שמגינות עליה — כל מקרה קצה מכוסה, כולל הטריקיים:

```ts
// lib/dormitory/occupancy.test.ts
it('is active on the boundary dates (inclusive)', () => {
  expect(isActiveOn(A('2026-01-01', '2026-12-31'), '2026-01-01')).toBe(true)
  expect(isActiveOn(A('2026-01-01', '2026-12-31'), '2026-12-31')).toBe(true)
})
it('is active for an open-ended assignment on/after the start', () => {
  expect(isActiveOn(A('2026-01-01', null), '2030-01-01')).toBe(true)
})
it('ignores ended assignments even inside the range', () => {
  expect(isActiveOn(A('2026-01-01', '2026-12-31', 'ended'), '2026-06-15')).toBe(false)
})
```

הדפוס הזה — הפרדת הלוגיקה מהמסד + בדיקה לכל מקרה קצה — חוזר בכל שמונת
המודולים החדשים (כספים, תחזוקה, הסעדה, מסמכים...). זו רמת עבודה של צוות
מנוסה.

## 4. מה נשאר

### מודולים (הישורת האחרונה)
1. **תורמים (Sponsors)** — כרטיס תורם, התחייבויות מול תקבולים. לזכור את סחיפת seed 002 שתועדה — ההרשאות ייזרעו במיגרציה עצמה.
2. **אבטחה** — יומן אירועים ומורשי כניסה. שניהם מתוכננים למחר לפי סיכום המפתח.

### תשתיות רוחביות שטרם נבנו (שלבים 3–4 בתוכנית האב)
3. **התראות ותזכורות** — עדיין אפס. עכשיו זה הפער הבולט ביותר: יש במערכת תאריכי יעד (משימות), מועדי חיוב (כספים), מעקבים (מרפאה/פסיכולוג) ותוקפי מסמכים — ואף אחד מהם לא מתריע מעצמו.
4. **העלאת קבצים אמיתית** — הבהרה חשובה: מודול "מסמכים" החדש הוא **רישום** מסמכים (מטא-דאטה: סוג, תוקף, למי שייך). בדקתי — אין עדיין אינטגרציית Supabase Storage, כלומר אי אפשר להעלות את הקובץ עצמו (סריקה/PDF). זה נשאר פער אמיתי.

### השלמות והקשחה
5. **סמסטרים ולוח שנה אקדמי** (שלב 2 בתוכנית) — לא טופל היום.
6. **השלמות כספים** — קבלות PDF, דוח גבייה, החזרים.
7. **ולידציית Zod ל-endpoints הישנים**, טיפוסים אוטומטיים מהמסד, ניטור שגיאות (Sentry), E2E מלא בענן.
8. **הכנה להשקה** — עדכון מדריכי משתמש, הדרכות, ייבוא נתוני אמת.
